Tris et rangs¶

Définition¶

On dfinit le problme du tri comme:

Entre: une squence de valeurs $a_1, a_2, ..., a_n$

Sortie: une permutation $\sigma$ (un rarrangement) telle que la squence $$a_{\sigma(1)}, a_{\sigma(2)}, ..., a_{ \sigma(n)}$$ soit ordonne.

Clé de tri¶

On parle de cl de tri pour dsigner l'lment sur lequel l'ordre est mis en place.

  • Type de cl usuelles: nombre, chaine, liste
  • Cl dfinie par l'utilisateur. Ex. pour une chaine
    • ordre lexicographique (par dfaut)
    • nombre de caractres pour une chaine
    • nombre de consonnes puis nombre de voyelles
    • mois de l'anne: 'Janvier' < 'Fvrier'

  • Cl simple dans une structure plus complexe.

    Ex.: Tuple (JJ, MM, YYYY)

Le langage de programmation fait donc des choix de cls par dfaut.

Il revient au programmeur de les adapter si besoin est, et de dfinir sa propre de comparaison.

Classification des algorithmes de tri¶

Dfinition: Tri en place ou non

Un algorithme de tri est dit en place lorsque jamais plus d'un nombre constant d'lments est stock hors du tableau quelque soit sa taille.

Dfinition: Tri stable

Un algorithme de tri est dit stable lorsque l'ordre des lments ayant une cl identique est maintenu.

On peut rendre stable un algorithme de tri en introduisant une cl secondaire.

Dfinition: Tri par comparaison ou non (voir TP)

  • Un algorithme de tri peut utiliser des comparaisons "<"
  • ou non...

Dfinition: Tri incrmental ou non

  • Un algorithme de tri est incrmental si il est succeptible de traiter les donnes une par une sans disposer du tableau en entier.

Tri par sélection¶

Caractéristiques: Tri par sélection¶

Il est:

  • en place
  • stable
  • par comparaison
  • non incrmental

Analyse du Tri par sélection¶

Terminaison: OK

Correction de l'algorithme par invariant:

Avant:

  • T[1..i-1] est tri
  • Pour tout j=i..n, T[j] >= T[i-1]

Aprs:

  • T[1..i] est tri
  • Pour tout j=i+1..n, T[j] $\geq$ T[i]

A la fin:

  • T[1..n-1] est tri
  • T[n] $\geq$ T[n-1]

=> T est donc tri !

Efficience de l'algorithme

Nombre d'changes ?

Pour i fix, 1 change.

Nombre total d'changes: n-1

Nombre de comparaisons ?

Pour i fix, n-i comparaisons.

  • i = 1 -> n - 1 comparaisons
  • i = 2 -> n - 2 comparaisons
  • ...
  • i = n - 2 -> 2 comparaisons
  • i = n - 1 -> 1 comparaison

Nombre total de comparaisons: $$1+2+3+...+(n-2)+(n-1) = \frac{n*(n-1)}{2} \sim n^2$$

def tri_selection(T):
    n = len(T)
    for i in range(0, n-1):
        imin = i
        for j in range(i+1, n):
            if T[j] < T[imin]:
                imin = j
        T[i], T[imin] = T[imin], T[i]
    return T

import random

def estTrie(L, deb=0, fin=None):
    if fin is None:
        fin = len(L)
    for i in range(1, fin):
        if L[i-1] > L[i] :
            return False
    return True

def tableau(n, min=0, max=100):
    return [random.randint(min, max) for _ in range(n)]

Tri par insertion¶

Tri par insertion (démo)¶

source: Tri par insertion (Wikipdia)

Caractéristiques: Tri par insertion¶

Il est:

  • en place
  • stable (car inf. strict)
  • par comparaison
  • incrmental

Terminaison de l'algorithme: OK

Correction de l'algorithme par invariant:

  • Avant: T[1..i-1] est tri
  • Aprs: T[1..i] est tri
  • A la fin (i = n), T est donc tri !

Efficience de l'algorithme

Le nombre de comparaisons est proportionnel au nombre d'changes.

  • Pour i fix, on distingue plusieurs cas:

    • 2 comparaisons si T[i] $\geq$ T[i-1]
    • 2*i comparaisons si T[i] = cl $<$ T[j] pour tout j=1..i-1.
    • i= 2*i/2 comparaisons en moyenne

Nombre total de comparaisons: $$2n \leq nbComp(n) \leq 2 n^2$$

def tri_insertion(T):
    n = len(T)
    for i in range(1, n):
        cle = T[i]
        j = i - 1
        while j >= 0 and T[j] > cle:
            T[j+1] = T[j]
            j = j - 1
        T[j+1] = cle 
    return T
